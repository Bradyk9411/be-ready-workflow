Be_Ready_AI 대시보드

정기모임 참석자를 위한 지속적인 바이브코딩 실행과 인사이트 공유 시스템.
① 실시간 동기화 ② 출석·참석률 ③ 단계 자동 보정 ④ CS센터·알림 🔒 프라이버시
📡
① 실시간 프로젝트 업데이트 동기화
Edge Function · sync-update
🎯문제정의

에이전트가 직접 데이터를 입력하는 워크플로우를 경험하고, 노션 데이터 수동 업데이트에서 탈피.

🛠사용 툴
참여자 AI 에이전트 be_ready_ai.md · /be-ready Supabase weekly_updates (append-only · RLS) DB 웹훅 pg_net Edge Function sync-update Notion API
🔄 워크플로우 — 약 4초
1
참여자: "대시보드 업데이트" (또는 /be-ready)
각자의 AI가 호출됨 — 사람이 노션을 열 필요 없음
2
내 AI가 PRD·진행을 읽고 정리
진행 단계 · 이번 주 한 줄 · 할 일 목록을 {key, name, status}로 도출
3
올리기 전 확인 — "이렇게 올릴게요, 맞나요?"
내가 "응" 하면 전송, "아니"면 고쳐서 다시 확인
4
Supabase로 전송 → DB 웹훅이 Edge Function 깨움
쓰기 전용 키로 weekly_updates에 INSERT → 트리거가 sync-update 호출
5
sync-update가 노션에 기록
① 닉네임 ↔ 멤버 명단 매칭 → ② 프로젝트 단계 갱신 → ③ 활동 로그 새로 쌓기 → ④ 할 일 4칸 정리(앞으로·진행중·대기·완료) → ⑤ "이번 주 현황" 숫자 재계산
✓
4초 뒤 대시보드에 자동 반영 ✅
피드 · 도움 필요 · 갤러리 · 펄스가 동시에 갱신
⌨️
커스텀 커맨드 /be-ready — 참여자가 본인 AI에 be_ready_ai.md를 한 번 붙여넣으면 만들어지는 슬래시 커맨드예요. 실행하면 그 AI가 PRD·진행을 읽고 → 올릴 내용을 도출 → 확인받고 → Supabase로 전송하는 전 과정을 대신합니다. 매번 양식을 외울 필요 없이 "대시보드 업데이트" 한 마디면 끝.
🛡️
유령 데이터 방지 — 닉네임이 멤버 명단에 없으면 노션에 아무것도 안 만들고 보류합니다. 아무나 남의 이름으로 글을 흘려넣을 수 없어요.
🔑
중복 없는 할 일 — 작업마다 고정 key로 식별(dedup). 문구를 다듬어도 같은 Task가 갱신될 뿐, 새로 생기지 않습니다.
🗓️
② 정기모임 출석·참석률 자동 집계
Edge Function · recompute-attendance · cron 15분
🎯문제정의

참석자 현황 및 CRM 시스템 구축을 통해 대시보드 현황판 자동화.

제약: 멤버 CRM(본명·연락처 등)은 비공개여야 한다. 공개 콜아웃에는 닉네임·숫자만.

🛠사용 툴
Notion 정기모임(회차) DB 멤버 CRM relation · 비공개 Notion 롤업 Count all Edge Function recompute-attendance pg_cron 15분 주기
🔄 워크플로우 — 15분마다 자동
1
운영자: 회차 DB에 참석/노쇼 체크
참석자·노쇼를 멤버 CRM과 관계(relation)로 연결
2
15분마다 recompute-attendance 자동 실행
pg_cron이 사람 없이 주기적으로 깨움
3
멤버별 가입후 모임 수 · 참석률(%) 갱신
집계 anchor의 전체 모임 수를 기준(분모)으로 자동 계산
✓
"다음 정기모임" 콜아웃 갱신
참석 확정 인원 · 참석률 · D-day를 CRM 미노출로 안전하게 공개
🐞
실제로 잡은 버그 — 참석 인원을 관계 "개수"로 세다 오집계가 났는데, 노션 롤업 Count all로 바꿔 해결했어요. (자동화는 분모·집계 방식이 핵심)
🩹
③ 프로젝트 단계 자동 보정
Edge Function · reconcile-stages · cron 하루 1회 (09:30 KST)
🎯문제정의

프로젝트 진행 상황에 따른 자동화 시스템 구축.

원칙: 멱등(변한 것만 PATCH), 사람의 명시적 선택은 존중(배포·완성, 활성 기획중은 건드리지 않음).

🛠사용 툴
Notion Project · Note DB Edge Function reconcile-stages pg_cron 하루 1회
🔄 워크플로우 — 하루 1회
1
매일 09:30 reconcile-stages 자동 실행
2
프로젝트별 최신 노트 신호 읽기
도움 필요 여부 · 막힌 점 · 마지막 업데이트 날짜를 확인
3
규칙대로 단계 보정 판단
도움필요 → 막힘 · 해결 → 개발중 · 14일 무업데이트 → 중단 · 중단 후 새 노트 → 재활성
✓
변한 것만 PATCH (멱등)
바뀐 프로젝트만 갱신 — 그대로면 아무 일도 안 일어남
⚖️
자동화가 넘지 않는 선 — 멤버가 직접 올린 배포·완성이나 활성 기획중은 자동 보정이 절대 덮어쓰지 않습니다. 자동은 어디까지나 '방치된 어긋남'만 정리.
🛟
④ CS센터 + 카카오 알림
공개 폼 → CS 접수 DB · /cs 접수 처리 · Edge Function cs-notify · cron 매일 09시
🎯문제정의

노션에 접속하지 않아도 실시간 CS 문의 확인 자동화.

제약: 개인 민원이 섞이므로 접수함은 비공개로 보호.

🛠사용 툴
Notion 공개 폼 비로그인 CS 접수 DB 비공개 운영자 /cs 접수 처리 Edge Function cs-notify 카카오 API 토큰 자동 갱신
🔄 워크플로우
1
참여자: 공개 폼 제출 (로그인 불필요)
제목 · 유형(건의/오류) · 내용 · 닉네임 · 첨부
2
CS 접수 DB에 행 생성 (직결)
폼이 곧 데이터 — 사이에 사람이 옮기는 단계 없음
3
매일 09시 cs-notify 자동 실행
새 접수가 있으면 운영자 카톡으로 다이제스트 전송 + 카카오 토큰 자동 갱신
✓
운영자: /cs 접수 처리
상태 · 우선순위 · 처리 메모를 같은 DB 안에서 정리하고 마감
🔒 한결같이 지킨 원칙 — 프라이버시 · 안전
자동화는 편하지만, 공개 대시보드라 "안전하게 열려있는지"가 더 중요했어요.
🙈 개인정보는 비공개
멤버 CRM(본명·연락처)은 관계로만 연결하고, 공개 뷰엔 닉네임만 노출. 공개 뷰는 공개=true 강제.
✍️ 쓰기 전용 키
외부에 나가는 건 append(쓰기) 전용 키뿐. 남의 글을 읽거나 고칠 수 없어요. service_role 키는 어디에도 배포 안 함.
👻 유령 데이터 차단
멤버 명단에 없는 닉네임으로 들어온 업데이트는 노션에 아무것도 만들지 않고 보류.
🗝️ 토큰은 금고에
Notion 토큰은 코드가 아니라 Supabase Vault에 보관, 서버 함수만 꺼내 씁니다.
